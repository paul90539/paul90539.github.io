建立 Hexo 環境

建立 Github repo

建立一個repo，名稱必需叫做username.github.io，username是你github的帳號
這為github預設的名稱，可以避免很多麻煩
建好repo後，去Settings->Pages將publish site打開其他人才看的到

安裝相關軟體

Hexo是依賴Node.js所開發的系統，而Git是在你要上傳至Github時會需要的軟體

  • Node.js
  • Git

建立部落格

1. 安裝 Hexo

npm install -g hexo-cli

2. 初始化部落格

hexo init blog_name //創建並初始化部落格，blog_name=你的部落格名稱
cd blog_name        //進入到部落格資料夾中
npm install         //安裝 Hexo
npm install hexo-deployer-git --save  //安裝 git 部署套件

3. 修改部落格資訊

修改 blog_name\_config.yml 中6~12行的內容

title:      部落格名稱
subtitle:   副標題
description:網站描述
keywords:   網站關鍵字(以逗號隔開)，方便 SEO 
author:     作者名稱
language:   您所使用的語言(zh-TW)
timezone:   留空以使用系統時間

以及第16行的url改成自己網站的連結

url: https://username.github.io/

接著修改部屬資訊，在文件中找到deploy:的標籤
repo是你要部屬的位置，branch是要部屬的目標分支

deploy:
  type: git
  repo: https://github.com/username/username.github.io.git
  branch: master

部署至 GitHub 與測試

測試

hexo s  //local端測試，http://localhost:4000/，也可以輸入 hexo server

部屬

hexo cl //清除靜態檔案，也可以輸入 hexo clean
hexo g  //建立靜態檔案，也可以輸入 hexo generate
hexo d  //部署至 Github Pages，也可以輸入 hexo deploy
